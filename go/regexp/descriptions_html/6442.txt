<br />
&#39;<a href="http://localhost/keyword/Macsyma">Macsyma</a>&#39; (Project <a href="http://localhost/keyword/MAC">MAC</a>’s  SYmbolic MAnipulator) は、1<a href="http://localhost/keyword/968%E5%B9%B4">968年</a>から1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>までMITの Project <a href="http://localhost/keyword/MAC">MAC</a> の一環として開発された数式処理システムであり、その後商用化された。世界初の数式処理システムで、初期のエキスパートシステムの1つであり、その様々なアイデアが後の Mathematica や Maple といったシステムに影響を与えた。<br />
<br />
== 開発 ==<br />
1<a href="http://localhost/keyword/968%E5%B9%B4">968年</a>7月、カール・エンゲルマン、（フロントエンド、数式表示、多項式算術）、ジョエル・モーゼス（簡約化、不定積分、ヒューリスティックス）によりプロジェクトが始まった。1<a href="http://localhost/keyword/971%E5%B9%B4">971年</a>までのプロジェクト責任者はマーティンで、モーゼスがその後<a href="http://localhost/keyword/10%E5%B9%B4">10年</a>間責任者を務めた。エンゲルマンと彼のスタッフは19<a href="http://localhost/keyword/69%E5%B9%B4">69年</a>に MITRE Corporation に戻っている. See also 。その後、様々な人々が開発に関与した。<br />
<br />
使用言語は Maclisp で、Maclisp を数値計算向きに改良する主な動機となった。Maclisp はPDP-6およびPDP-10が主なプラットフォームだったが、MulticsオペレーティングシステムやLISPマシンアーキテクチャでも動作した。<a href="http://localhost/keyword/Macsyma">Macsyma</a> は当時としては大規模なLISPプログラムの1つだった。<br />
<br />
=== 商用化 ===<br />
1<a href="http://localhost/keyword/979%E5%B9%B4">979年</a>、当時カリフォルニア大学バークレー校 (UCB) の教授だったリチャード・フェイトマンの要望で、MITは <a href="http://localhost/keyword/Macsyma">Macsyma</a> のコードの一時的ライセンスを提供した。これを使い、フェイトマンの研究室で Maclisp から派生させた  を使い、 上にすぐさま移植した。MITは、適切なライセンス条件の交渉が完了した際には現状の一時的ライセンスは破棄されるという条件で、UCBが VAX 版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> をカリフォルニア工科大学など約50の大学に配布することをしぶしぶ許可した。実際、後述するシンボリックスとの契約が成立した際に従来のライセンスは破棄された。するとシンボリックスはVAX製品が同社のLISPマシンと性能的に競合することからVAX版<a href="http://localhost/keyword/Macsyma">Macsyma</a>のライセンス提供を渋り、結局<a href="http://localhost/keyword/5%E5%B9%B4">5年</a>間VAX版のライセンスを提供しなかった。UCBはさらにサンのワークステーションなどMC68000を使ったシステムにも<a href="http://localhost/keyword/Macsyma">Macsyma</a>を移植した。同じころフェイトマンは、破棄された当初のライセンスを恒久化すべく働きかけていた。<br />
<br />
最終的に1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>、アメリカ合衆国エネルギー省 (DoE) はMITに対して National Energy Software Center (NESC) ライブラリにコピーをリリースさせるという条件と引き換えに、高い価格設定と再配布禁止というライセンス条件をMITが設定することを許可した。これはシンボリックスへの技術移転を保護することを意図したものだった（そのような制限は200<a href="http://localhost/keyword/2%E5%B9%B4">2年</a>ごろ撤廃された）。このいわゆる DOE <a href="http://localhost/keyword/Macsyma">Macsyma</a> はMITが Common Lisp の前身である  で書き直したもので、当時学界で主流だった Berkeley VAX Unix ではなく、人気のない VAX/VMS で動作した。DOE <a href="http://localhost/keyword/Macsyma">Macsyma</a> は後のオープンソースの Maxima の基礎となった。<br />
<br />
1<a href="http://localhost/keyword/981%E5%B9%B4">981年</a>、モーゼスとリチャード・パヴェル（MIT職員で、<a href="http://localhost/keyword/Macsyma">Macsyma</a>を工学や科学に適用することを提案した）は <a href="http://localhost/keyword/Macsyma">Macsyma</a> を商用化するための会社を創業しようとした。パヴェルは <a href="http://localhost/keyword/Macsyma">Macsyma</a> を使った科学的論文を多数書いていた。そうした論文を手に、パヴェルとモーゼスは出資に興味を示したいくつかのベンチャーキャピタルを訪れた。契約が成立しそうになったころ、MITはMITの人間がMITでの開発で直接利益を得るべきではないと決定した。1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>初め、MIT は <a href="http://localhost/keyword/Macsyma">Macsyma</a> をアーサー・D・リトル (ADL) にライセンス供与し、同社が <a href="http://localhost/keyword/Macsyma">Macsyma</a> の仲買人となり、1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>後半には <a href="http://localhost/keyword/Macsyma">Macsyma</a> をシンボリックスにライセンス供与した。この間にADLによってモーゼスが締め出され、パヴェルがシンボリックスの<a href="http://localhost/keyword/Macsyma">Macsyma</a>部門のトップに就任することになった。シンボリックスは<a href="http://localhost/keyword/Macsyma">Macsyma</a>を独占することにはそれほど興味がなく、競合するLISPマシン企業  もライセンス供与を受けている。シンボリックスとADLの契約では、<a href="http://localhost/keyword/Macsyma">Macsyma</a>の売り上げの15%をロイヤリティとしてADLに支払うことになっていた。この法外なロイヤリティは、<a href="http://localhost/keyword/Macsyma">Macsyma</a> が絶対に売れるとMITとADLが考えていたことの表れである。シンボリックスではLISPマシンを売るのが本業と考えていたが、<a href="http://localhost/keyword/Macsyma">Macsyma</a> の開発も継続していた。<a href="http://localhost/keyword/Macsyma">Macsyma</a> およびそれを搭載したLISPマシンの売り上げは<a href="http://localhost/keyword/2%E5%B9%B4">2年</a>以内にシンボリックスの売り上げ全体の10%を占めるようになっていった。シンボリックス社内からは多くの抵抗があったが、19<a href="http://localhost/keyword/80%E5%B9%B4">80年</a>代初めから中ごろに <a href="http://localhost/keyword/Macsyma">Macsyma</a> をバークレーの Franz Lisp で書いたものがDECのVAXやサンのワークステーション向けにリリースされた。<br />
<br />
他社のコンピュータ向けにも <a href="http://localhost/keyword/Macsyma">Macsyma</a> が販売されるようになると、<a href="http://localhost/keyword/Macsyma">Macsyma</a> を搭載したシンボリックスのLISPマシンの売り上げが低下していった。市場自体は成長しているにも関わらず、1<a href="http://localhost/keyword/986%E5%B9%B4">986年</a>上半期の <a href="http://localhost/keyword/Macsyma">Macsyma</a> の売り上げは、1<a href="http://localhost/keyword/985%E5%B9%B4">985年</a>上半期の売り上げより低下した。数式処理という点では明らかに <a href="http://localhost/keyword/Macsyma">Macsyma</a> の方が優れていたが、そのころスティーブン・ウルフラムのSMPやウォータールー大学のMapleが売り上げを伸ばしていた。<br />
<br />
パヴェルはシンボリックスの <a href="http://localhost/keyword/Macsyma">Macsyma</a> 部門を1<a href="http://localhost/keyword/986%E5%B9%B4">986年</a>初めごろまで指揮していた。1<a href="http://localhost/keyword/986%E5%B9%B4">986年</a>後半にはリチャード・ペッティに引き継がせ、シンボリックス社内の衝突を避けるため、経営陣は <a href="http://localhost/keyword/Macsyma">Macsyma</a> の販売を減らす方針を採用した。<a href="http://localhost/keyword/Macsyma">Macsyma</a>部門の従業員数は削減されたが、営業部門は強化し、顧客が求める機能を開発することに集中するようにした。例えば、グレブナー基底を求めるアルゴリズムは1<a href="http://localhost/keyword/970%E5%B9%B4">970年</a>代にMITで発展したが、1<a href="http://localhost/keyword/987%E5%B9%B4">987年</a>まで製品版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> にそれが搭載されることはなかった。1<a href="http://localhost/keyword/987%E5%B9%B4">987年</a>、<a href="http://localhost/keyword/Macsyma">Macsyma</a> の年間売り上げはほぼ倍増した。マニュアルやオンラインヘルプが改良され、コマンド名をさらに覚えやすくし、<a href="http://localhost/keyword/Macsyma">Macsyma</a> は格段に使いやすくなっていった。ペッティはシンボリックス経営陣に対して、<a href="http://localhost/keyword/Macsyma">Macsyma</a> はハードウェア部門の戦略とは切り離して出資されるべき戦略事業単位だと主張した。しかし、シンボリックスはその後も <a href="http://localhost/keyword/Macsyma">Macsyma</a> 部門の人員を削減している。シンボリックスは多大な赤字を出しているハードウェア部門の赤字補填のドル箱として <a href="http://localhost/keyword/Macsyma">Macsyma</a> を使おうとした。<br />
<br />
<a href="http://localhost/keyword/Macsyma">Macsyma</a> の最大の弱点は数値解析能力の低さだった。数式処理は難しいタスクだが、数値解析はより大きな科学技術計算市場に参入する際に重要となる。MIT版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> は数値計算ライブラリ IMSL とリンクしていたが、シンボリックス版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> ではこのリンクは難しかった（LISPマシン向けのIMSLがない）。シンボリックスのLISP開発者は数値解析を古い技術だと信じていて、LISPの用途としては重要でないと考えていたため、その方面の強化を怠っていた。PC版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> の倍精度演算は、FORTRANの6倍の時間がかかった。また行列をリストのリストとして実装していたため、重要なアルゴリズムの性能低下の要因となっていた。<a href="http://localhost/keyword/Macsyma">Macsyma</a> にはLU分解のような数値線形代数の基本アルゴリズムも備わっていなかった。<br />
<br />
1<a href="http://localhost/keyword/987%E5%B9%B4">987年</a>から19<a href="http://localhost/keyword/88%E5%B9%B4">88年</a>にかけて、Gold Hill Lisp を使ったPC版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> をリリースしようとした。一般的なコンピュータ向けにLISPコンパイラを開発することはシンボリックスにとって本業であるLISPマシンと競合する相手を作るようなもので、シンボリックス経営陣はそのプロジェクトをやめさせた。しかし、プロジェクトは経営陣には無断で続行された。ところがこの Gold Hill Lisp は非常に不安定で、アーキテクチャに問題があるためバグ修正も難しかった。これが <a href="http://localhost/keyword/Macsyma">Macsyma</a> にとっては致命傷となった。19<a href="http://localhost/keyword/88%E5%B9%B4">88年</a>中ごろにMathematicaのMacintosh版が登場したとき、PC版<a href="http://localhost/keyword/Macsyma">Macsyma</a>を対抗してリリースすることができなかった。Windows版<a href="http://localhost/keyword/Macsyma">Macsyma</a>は、シンボリックスが開発した CLOE Lisp を使って19<a href="http://localhost/keyword/89%E5%B9%B4">89年</a>8月にリリースされた。しかしそのころ <a href="http://localhost/keyword/Macsyma">Macsyma</a> 部門の人員はあまりに少なく、Mathematicaが備えていたグラフィックス描画機能、ノートブック・インタフェース、数値演算機能などに対抗できるものを実装できなかった。<br />
<br />
19<a href="http://localhost/keyword/89%E5%B9%B4">89年</a>、シンボリックスは製品戦略の失敗から事業を整理統合する必要に迫られた。ペッティは <a href="http://localhost/keyword/Macsyma">Macsyma</a> 部門を独立させようとしたがMITから資金協力を得られなかった。19<a href="http://localhost/keyword/88%E5%B9%B4">88年</a>末、ペッティは経営陣にソフトウェア専業への移行を進言したが、受け入れられなかった。そこで新たな企業を創業するためペッティはシンボリックスを退社した。<br />
<br />
=== <a href="http://localhost/keyword/Macsyma">Macsyma</a>, Inc. ===<br />
1<a href="http://localhost/keyword/992%E5%B9%B4">992年</a>、シンボリックスの創業者の1人（会長）とリチャード・ペッティ（社長）が <a href="http://localhost/keyword/Macsyma">Macsyma</a>, Inc. を創業。シンボリックスから <a href="http://localhost/keyword/Macsyma">Macsyma</a> の権利を買い取った。市場は急激に成長していたが、<a href="http://localhost/keyword/Macsyma">Macsyma</a> の売り上げは1<a href="http://localhost/keyword/991%E5%B9%B4">991年</a>から1<a href="http://localhost/keyword/992%E5%B9%B4">992年</a>初めにかけても急激に落ち込んでいた。数式処理ソフトウェアにおける<a href="http://localhost/keyword/Macsyma">Macsyma</a>のシェアは、1<a href="http://localhost/keyword/987%E5%B9%B4">987年</a>には70%だったものが、1<a href="http://localhost/keyword/992%E5%B9%B4">992年</a>には1%となっている。19<a href="http://localhost/keyword/93%E5%B9%B4">93年</a>に市場の成長がおさまったころには、Mathematica と Maple がほとんどを占めるようになっていた。19<a href="http://localhost/keyword/90%E5%B9%B4">90年</a>代を通して、<a href="http://localhost/keyword/Macsyma">Macsyma</a>, Inc. の開発要員は競合他社の4分の1から8分の1だった。<br />
<br />
1<a href="http://localhost/keyword/995%E5%B9%B4">995年</a>初め、様々な改良を施した <a href="http://localhost/keyword/Macsyma">Macsyma</a> 2.0.5 をリリース。Wester による数式処理システムの大規模な評価によると、<a href="http://localhost/keyword/Macsyma">Macsyma</a> 2.0.5 は Maple より10%、Mathematica より15%性能がよいとされている Wester&#39;s 1995 review and 1999 review。<a href="http://localhost/keyword/Macsyma">Macsyma</a> 2.0.5 の数値演算性能は相変わらず低かったが、数値解析および線形代数の<a href="http://localhost/keyword/%E3%83%AB%E3%83%BC%E3%83%81%E3%83%B3">ルーチン</a>は大幅に強化されていた。1<a href="http://localhost/keyword/996%E5%B9%B4">996年</a>には LAPACK を採用し、数値線形代数の性能も強化された。<br />
<br />
しかし開発チームの人員の少なさから、性能上の優位を継続的に維持することはできなかった。市場シェアは2%を越えることはなかった。19<a href="http://localhost/keyword/99%E5%B9%B4">99年</a>、<a href="http://localhost/keyword/Macsyma">Macsyma</a>, Inc. はシンボリックスも買収した企業 Tenedos LLC に買収された。この企業が <a href="http://localhost/keyword/Macsyma">Macsyma</a> を販売することはなかったが、シンボリックスが<a href="http://localhost/keyword/Macsyma">Macsyma</a>の販売を継続している。<br />

== 入手可能なバージョン ==<br />
Maxima は1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>版の DOE <a href="http://localhost/keyword/Macsyma">Macsyma</a> をベースとしたGPLライセンス版であり、後に William Schelter が Common Lisp で書き直し強化を施していった。今も盛んに開発が続いている。GNU/Linux、Windows、Mac OS X などのシステム向けのGUIも備えた実行ファイル版がダウンロード可能である。1<a href="http://localhost/keyword/982%E5%B9%B4">982年</a>以降の商用版 <a href="http://localhost/keyword/Macsyma">Macsyma</a> での強化をほとんど含んでいないが、オープンソースであるため様々な機能が追加されてきた。そのため <a href="http://localhost/keyword/Macsyma">Macsyma</a> と Maxima には非互換があるが、<a href="http://localhost/keyword/Macsyma">Macsyma</a>の代数言語で書かれたものはほぼ変更なく両方のシステムで実行可能である。<br />
